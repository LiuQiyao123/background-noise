# 底噪 V2 设计系统 — 地下现场美学宣言

> **版本**: V2.0 · 风格化重构  
> **设计定位**: 给音乐爱好者的记忆社交平台  
> **灵感来源**: chopin/bnoise iOS · 地下 Livehouse 现场 · 独立音乐视觉文化  
> **设计师注**: 我组过乐队，在 MAO 后台调过音，在 VOX 门口发过传单。我知道乐迷要什么——不是精致的包装，是现场的余温。

---

## 🎸 第一章：设计哲学

### 1.1 核心隐喻

> **「底噪」= 现场录音里那层挥之不去的背景噪声**

它不是缺陷，是**存在的证明**。就像：
- 演出结束后耳朵里的嗡鸣
- 排练室墙角的电流声
- 磁带 B 面的沙沙声
- 票根边缘的毛糙感

我们的设计要捕捉这种**不完美的真实感**。

### 1.2 设计原则（乐迷视角）

| 原则 | 解释 | 反例 |
|------|------|------|
| **像票根，不像海报** | 票根是私人的、有使用痕迹的；海报是官方的、完美的 | 避免光鲜渐变、完美圆角 |
| **像手账，不像 UI** | 手账有撕边、有涂改、有贴纸；UI 是标准化的 | 避免过度统一的组件库感 |
| **像现场，不像精修** | 现场有噪点、有暗角、有过曝；精修图没有灵魂 | 避免过度平滑、过度清晰 |
| **像暗号，不像广播** | 暗号只有圈内人懂；广播是大众传播 | 避免泛社交化的设计语言 |

### 1.3 情绪板（文字版）


## 🎨 第二章：视觉语言升级

### 2.1 色彩系统 2.0

**从「暖纸色」升级为「现场光」**

#### 主色系（暗场模式）

| 令牌 | 旧值 | 新值 | 用途 |
|------|------|------|------|
| `bg-stage` | `#F5EDE0` | `#0a0a0c` | 舞台黑场（主背景） |
| `bg-spotlight` | `#E0D6C2` | `#1a1a20` | 聚光灯区（卡片） |
| `bg-foam` | `#BCBAB5` | `#2a2a32` | 泡沫垫（次级元素） |
| `text-ink` | `#121212` | `#f2f2f4` | 主文字（反白） |
| `text-stage` | `#8C8A84` | `#8b8b96` | 舞台灰（次要文字） |

#### 强调色（现场光）

| 令牌 | 值 | 来源 | 用途 |
|------|-----|------|------|
| `accent-red` | `#e85d4c` | 舞台侧屏红灯 | 主按钮、选中态 |
| `accent-amber` | `#f2c82e` | 吉他 amp 指示灯 | 警告、高亮 |
| `accent-purple` | `#9b6b9e` | 烟雾机紫光 | 氛围标签 |
| `accent-blue` | `#6b8cae` | 调音台背光 | 信息提示 |

#### 功能色（Vibe 三维）

```
乐队表现   → 琥珀色  #c9a227  (像吉他音箱的暖光)
音响效果   → 灰蓝色  #6b8cae  (像调音台的冷光)
现场氛围   → 灰紫色  #9b6b9e  (像舞台烟雾的紫光)
```

### 2.2 字体系统 2.0

**从「标准字体」升级为「演出海报感」**

#### 字体栈

```css
/* 中文：清晰克制，像演出日程表 */
--font-zh: "Noto Sans SC", "Source Han Sans CN", system-ui, sans-serif;

/* 英文/数字：带一点工业感 */
--font-en: "DM Sans", "Space Grotesk", "Segoe UI", sans-serif;

/* 特殊场景：等宽字体像后台技术清单 */
--font-mono: "JetBrains Mono", "Fira Code", monospace;

/* 标题/Logo：衬线体像老海报 */
--font-display: "Noto Serif SC", "Source Han Serif CN", serif;
```

#### 字号层级（移动端基准）

```
Display (艺人名/标题):  28px / 700 / line-height 1.2
Title (页面标题):       20px / 600 / line-height 1.3
Body (正文):            15px / 400 / line-height 1.5
Caption (元数据):       13px / 400 / line-height 1.4
Micro (标签/角标):      11px / 500 / line-height 1.2

/* 特殊：一句话记忆 */
Memory-Hook:            17px / 600 / line-height 1.4 / 斜体
```

### 2.3 质感系统（核心差异化）

**从「扁平」升级为「有触感的数字」**

#### 2.3.1 纸张纹理（Paper Grain）

```tsx
// SVG noise filter + fiber lines
<filter id="paper-grain">
  <feTurbulence type="fractalNoise" baseFrequency="0.65" numOctaves="3" />
  <feColorMatrix type="saturate" values="0" />
</filter>

// 叠加方式
background-image: url("data:image/svg+xml,...");
opacity: 0.08;  // 极轻，不抢戏
```

**使用场景**:
- 票根卡片背景
- 记录发布页
- 个人主页统计卡

#### 2.3.2 混凝土纹理（Concrete）

```tsx
// 暗色系页面专用
base: #1a1a20
speckles: random black dots (opacity 0.05-0.15)
fibers: thin lines (8-14条)
light-spots: white specks (5处)
vignette: radial gradient (opacity 0.22)
```

**使用场景**:
- 演出详情页背景
- 登录/注册页
- 设置页

#### 2.3.3 撕裂边缘（Torn Edge）

```tsx
// SVG path 生成不规则边缘
// 参数：seed, jagAmplitude(3-5px), jagFrequency(18-22px)
clipPath: polygon(
  0% 2%, 5% 0.5%, 10% 3%, 15% 1%, 20% 4%, ...
  100% 98%, 95% 99.5%, 90% 97%, ...
)
```

**使用场景**:
- 票根卡片
- 记忆墙标签
- 特殊状态卡片（如盲盒）

#### 2.3.4 条形码/二维码装饰

```tsx
// 基于 seed 生成伪随机条形码
bars: [{x, width}, ...]
color: #121212 (暗色模式反白)
```

**使用场景**:
- 票根卡片左侧存根
- 分享图底部装饰
- 用户 ID 视觉化

### 2.4 动效系统

**从「标准过渡」升级为「现场感动效」**

| 场景 | 动效 | 时长 | 缓动 |
|------|------|------|------|
| 卡片 Hover | 上浮 2px + 阴影加深 | 150ms | ease-out |
| 按钮按下 | 下沉 1px + scale 0.98 | 80ms | ease-in |
| 点赞 | 图标 scale 1.2 → 1.0 + 粒子飞散 | 200ms | spring(0.6, 12) |
| 页面切换 | 淡入 + 轻微上移 | 250ms | ease-out |
| 盲盒揭晓 | 翻转 180deg + 模糊消散 | 400ms | ease-in-out |
| 匹配成功 | 脉冲 glow + 震动 | 300ms | ease-out |

**关键细节**:
- 所有动效必须有**物理感**（重量、惯性）
- 避免纯线性动画
- 重要交互（如点赞）加**触觉反馈**（如果平台支持）

## 🧩 第三章：组件系统重构

### 3.1 票根卡片 (TicketStubCard) — 核心组件

**设计目标**: 像从口袋里掏出的旧票根，有使用痕迹

```tsx
<TicketStubCard
  show={showData}
  variant="standard"  // standard | compact | mini
  tilt="random"       // random | none | fixed(deg)
  grainIntensity={0.12}
  tornEdge={true}
/>
```

**结构拆解**:

**CSS 关键点**:
```css

  position: absolute;
  inset: -7px;
```

### 3.2 记忆墙标签 (MemoryPill)

**设计目标**: 像贴在墙上的便签，可以撕下来

```tsx
<MemoryPill
  label="跳水那一刻"
  count={12}
  selected={false}
  variant="standard"  // standard | stamp | handwritten
/>
```

**变体**:
- **standard**: 圆角 pill，border + bg
- **stamp**: 圆形印章效果，红色边框
- **handwritten**: 手写字体 + 不规则边缘

### 3.3 同场盲盒 (BlindBoxCard)

**设计目标**: 像神秘包裹，揭晓时有仪式感

```tsx
<BlindBoxCard
  memoryHook="吉他 solo 时全场大合唱"
  matchLevel={3}
  similarityScore={87}
  onReveal={handleReveal}
  onDismiss={handleDismiss}
/>
```

**揭晓动效**:
```tsx
// 1. 卡片翻转 180deg
// 2. 模糊滤镜消散 (blur(10px) → blur(0))
// 3. 用户信息淡入
// 4. 匹配度徽章脉冲 glow
```

### 3.4 Vibe 评分器 (VibeRating)

**设计目标**: 像调音台推子，有机械感

```tsx
<VibeRating
  band={4}
  sound={5}
  atmosphere={4}
  interactive={true}  // 可调节
  showLabels={true}
/>
```

**视觉特征**:
- 每个维度有独立颜色（琥珀/灰蓝/灰紫）
- 滑块有刻度感（1-5  discrete steps）
- 当前值有微光 glow

### 3.5 Section Header

**设计目标**: 像演出海报的标题区

```tsx
<SectionHeader
  title="发现同好"
  subtitle="DISCOVER · 撞见同类"
  showHash={true}  // 显示 # 号
  variant="poster"  // poster | minimal | handwritten
/>
```

**变体**:
- **poster**: 大衬线体 + #号 + 英文副标
- **minimal**: 纯中文，无装饰
- **handwritten**: 手写字体（需额外字体文件）

> 这不是一个「好看」的 App，这是一个**有态度**的 App。
>
> 当用户打开它，应该感觉到：
> - 像走进熟悉的 Livehouse

>
