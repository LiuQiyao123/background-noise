# 底噪 UI 设计规范

> 版本：MVP v1 · 文档交付（指导设计与研发，非最终实现稿）  
> 关联：[Product-Spec.md](../Product-Spec.md) · [MVP-AND-SOCIAL-DECISIONS.md](./MVP-AND-SOCIAL-DECISIONS.md)  
> 目标平台：移动端优先（iOS / Android）；当前 Web 前端可作联调壳

## 1. 设计定位与原则

### 1.1 一句话

像 **Live House 关灯后的余温**——暗、躁、干净，不喧宾夺主。

### 1.2 情绪与气质

| 维度 | 方向 |
|------|------|
| 情绪 | 现场感、共鸣、私密可藏 / 公开可晒 |
| 参考 | 独立音乐场景、手环与票根、弱霓虹、轻微颗粒噪点（呼应产品名「底噪」） |
| 避免 | 泛社交信息流、高饱和渐变、官方海报风、过多营销徽章 |

### 1.3 设计原则

1. **内容优先**：Repo 与「一句话记忆」是视觉焦点，导航与装饰退后。
2. **默认可藏**：私密态有明确、安心的视觉区分（锁标 / 灰阶标签），降低「不敢记」心理成本。
3. **共鸣可发现**：记忆墙、同标签聚合是「共同记忆」的主视觉钩子。
4. **信噪比优先**：单屏信息有主次，避免同时强调过多 KPI。

### 1.4 情绪板（文字版）

- **主背景**：接近黑场的暖灰（ `#0a0a0c` → `#0c0b0f` ），非纯 `#000`。
- **强调**：现场红 `#e85d4c`，像舞台侧屏与手环荧光，用量约 8–12% 像素面积。
- **质感**：极弱噪点 + 暗角；卡片像「黑胶封套」而非玻璃拟物。
- **字体气质**：中文清晰克制；数字与日期略紧，带一点演出日程感。

## 3. 核心组件规范

每个组件：**用途 → 结构 → 状态 → 文案规则**

### 3.1 BottomTab（底部导航）

**用途**：全局一级导航。Tab：**现场** | **发现** | **记录** | **我的**。

**结构**：
- 固定底栏，高 `64px + safe-area`；顶部分割线 `color-border`。
- 每项：图标（24px）+ 文案（micro）；「记录」可用略大图标或圆形强调底（非必须 FAB）。

**状态**：

| 状态 | 视觉 |
|------|------|
| 默认 | `text-secondary` |
| 选中 | `text-accent` + 顶部 2px accent 条或图标 fill |
| 按下 | opacity 0.7 |

**规则**：最多 4 项；不在 Tab 放消息红点（MVP 无私聊）。

---

### 3.2 SearchBar

**用途**：搜索乐队、场地、城市。

**结构**：圆角 `radius-md` 输入条 + 右侧搜索按钮（Ghost）。

**占位符**：`搜索乐队、场地、城市…`

**状态**：默认 / 聚焦（border-focus）/ 禁用。

---

### 3.3 ShowCard（演出条目卡片）

**用途**：列表中的单场演出入口。

**结构**：
```
┌─────────────────────────────────────┐
│ [可选 48×48 封面]  艺人名（Title）    │  N 条现场
│                    3月15日 · MAO · 北京│  (tag)
└─────────────────────────────────────┘
```

**状态**：默认 / 按下（opacity 0.92）。

**规则**：艺人名一行截断；日期格式 `M月D日 · 场馆 · 城市`。

---

### 3.4 ShowHero（条目详情顶区）

**用途**：演出条目详情页顶部，对应 Spec「海报毛玻璃」。

**结构**：
- 高度约 160–200px；背景为 UGC 最高赞图 blur，无图时用 `bg-hero` 渐变 + 噪点。
- 前景左下：艺人名 Display；次行：日期时间 + 场馆 + 城市。
- 底部渐变保证文字对比度 ≥ 4.5:1。

**状态**：有封面 / 无封面（占位渐变）。

---

### 3.5 MemoryPill（记忆墙标签）

**用途**：条目详情「记忆墙」横向滚动，**共同记忆核心组件**。

**结构**：水平 `scroll-snap`；pill 内文案 `短句 (count)`，如 `跳水那一刻 (12)`。

**状态**：

| 状态 | 视觉 |
|------|------|
| 默认 | `bg-elevated` + `border` |
| 选中 | `border-accent` + `accent-muted` 底 + `text-accent` |
| 按下 | scale 0.98 |

**规则**：短句 ≤20 字；过长省略号；「全部」作为第一项。

---

### 3.6 RepoCard（现场记录卡片）

**用途**：条目下 Repo 列表、发现页共鸣流、个人时间线。

**结构**：
```
┌─────────────────────────────────────┐
│ 「一句话记忆」              [私密?]  │
│ 正文摘要最多两行…                    │
│ 昵称 · ♥ 12 · 💬 3                   │
└─────────────────────────────────────┘
```

**状态**：公开 / 私密（右上角 PrivacyBadge）/ 按下。

**规则**：记忆句用中文弯引号「」；私密卡整体对比度略降（非禁用灰）。

---

### 3.7 VibeBlock（三维评分）

**用途**：记录发布页、条目统计区。

**结构**：
- **录入**：三个 range 或五星行；左侧维度名 + 右侧数值 1–5。
- **展示**：三列 stat，数字 + 标签（乐队 / 音响 / 氛围），可用 vibe 辅助色点缀数字。

**维度文案（固定）**：`乐队表现` | `音响效果` | `现场氛围`

---

### 3.8 PrivacyBadge

**用途**：标识 Repo 可见性。

| 值 | 文案 | 视觉 |
|----|------|------|
| 私密 | `仅自己` | 锁图标 + `color-private` |
| 公开 | `已公开` | `color-public` 弱强调 |

---

### 3.9 Button

| 类型 | 用途 | 视觉 |
|------|------|------|
| Primary | 写记录、登录、保存 | 满幅 accent 底，白字 |
| Ghost | 次要操作、筛选 | 描边 border |
| Danger | 退出登录、删除 | 描边 danger 或纯文字 danger |

**最小点击区域**：44×44px。

---

### 3.10 EmptyState

**结构**：简线图标（可选）+ 一句说明 + 可选 CTA（Primary/Ghost）。

**示例文案**：
- 无演出：`还没有演出条目，去搜索或创建`
- 无公开 Repo：`第一个写下你的现场记忆吧`

---

### 3.11 SegmentedControl

**用途**：即将 / 热门 / 全部；热门 / 最新。

**状态**：选中项 `accent-muted` 底 + accent 字；未选中透明。

## 5. 动效与微交互

| 场景 | 建议 |
|------|------|
| Tab 切换 | 选中指示条 150ms ease，无整页闪白 |
| MemoryPill 选中 | 背景色 120ms ease |
| 点赞 | 图标 scale 1.15 → 1.0，150ms；数字 +1 可选 |
| 列表进入 | 可选 20ms stagger fade（低优先级） |
| 页面切换 | 移动端 slide 或 cross-fade，≤ 250ms |

**避免**：大面积 parallax、过重弹簧动画。

---

## 6. 无障碍与可读性

- 正文与背景对比度 ≥ **4.5:1**（WCAG AA）；大字标题 ≥ 3:1。
- 可点击区域最小 **44×44px**。
- 不单独用颜色传达状态（私密/公开需图标或文案）。
- 表单控件需可见 `focus` 态（accent 描边）。
- 支持系统字体缩放；布局避免固定高度截断正文。

## 8. 与 MVP 产品决策对齐

依据 [MVP-AND-SOCIAL-DECISIONS.md](./MVP-AND-SOCIAL-DECISIONS.md)：

| 决策 | UI 体现 |
|------|---------|
| **默认私密** | 记录页默认「仅自己」；RepoCard / 详情有 PrivacyBadge |
| **记忆墙必上** | 条目详情独立区块 + MemoryPill 组件规范 |
| **无 POA** | 不出现验证戳、票根、勋章 UI |
| **排序无「已验证」** | Repo 列表仅「热门 / 最新」，无验证角标 |
| **社交 = 点赞 + 评论** | Repo 详情仅赞/评；无私信、无群入口 |
| **无 AI 扩写按钮** | 记录页不出现（V2 再加） |
